# 论文修改意见与章节调整计划

本文档用于持续维护论文修改背景、行文逻辑、章节级计划和待办事项。后续每次讨论或修改章节后，应同步更新本文档，方便在新的会话中快速恢复上下文。

## 1. 当前核心修改目标

论文原有叙事从“深度学习在眼底疾病诊断中重要”切入，容易被追问：在视觉大语言模型和多模态模型快速发展的背景下，为什么本文选择卷积神经网络而不是视觉大语言模型？

新的叙事应从临床任务和部署场景出发，而不是从深度学习方法本身出发：

1. 眼底成像是早期眼科疾病辅助筛查的重要依据。
2. 基于眼底图像的智能辅助诊断主要包括传统机器学习、视觉大语言模型和卷积神经网络三类路线。
3. 比较三类路线后，说明本文选择卷积神经网络的合理性：
   - 传统机器学习依赖人工特征设计，建模效率和表达能力有限。
   - 视觉大语言模型通常需要大规模数据和高算力支持，在医疗数据稀缺、隐私受限的实际部署场景中成本较高。
   - 卷积神经网络适合眼底图像视觉特征建模，具有轻量、高效和易部署等优势。
4. 在选择卷积神经网络后，再递进提出真实临床应用中的三个挑战：
   - 临床任务层面：单眼多疾病共存，标准单标签分类器易误判或漏检。
   - 医院部署层面：医疗图像标注成本高且数据共享受限，难以获取多机构数据联合训练；同时跨机构部署又会因设备、光照、人工处理流程差异产生域偏移，导致泛化性能下降。因此需要单域泛化（SDG），而不是默认采用多源 DG。
   - 社会隐私层面：医疗数据高度敏感，传统域适应方法需要同时访问源域和目标域数据，在多域暴露场景下存在隐私机制添加复杂、性能衰减严重等问题。
5. 方法贡献需要与三个挑战一一对应：
   - 针对多疾病共存：嵌入 Transformer 机制建模标签关联性，减少多标签识别中的误检漏检。
   - 针对多机构数据难联合训练与跨机构部署域偏移并存的问题：设计双分支引导单域泛化（SDG）框架，在多源 DG 前提难以满足时，仅利用单一源域学习可迁移的疾病相关表示，并结合 CAM 引导提供可解释性。
   - 针对跨域场景隐私防护：在单域泛化 SDG 训练过程中集成高斯差分隐私，通过动态梯度噪声注入，在中等隐私保护下性能接近基线。

## 2. 当前摘要逻辑

当前摘要已经调整为以下顺序：

1. 眼底成像的重要性。
2. 三类技术路线比较：传统机器学习、视觉大语言模型、卷积神经网络。
3. 选择卷积神经网络的理由。
4. 三个递进挑战：
   - 多疾病共存导致单标签分类器误判或漏检。
   - 多机构数据难联合训练，同时跨机构部署存在域偏移，导致泛化性能下降。
   - 传统域适应方法在多域暴露场景下面临隐私机制复杂和性能衰减问题。
5. 三个对应工作：
   - Transformer 标签关联建模。
   - 双分支引导 SDG 框架与 CAM 可解释性。
   - 单域泛化 SDG 训练中集成动态高斯差分隐私。
6. 保留原总结句：
   “综合以上，我们搭建了一个具有隐私保护的多标签眼病识别可解释单域泛化框架。在实验部分，通过完整的消融实验分析证明该框架有效可靠，跨域表现优异，且隐私模式下性能衰减可控。”

## 3. 全文需要统一的叙事顺序

全文宏观逻辑应统一为：

眼底成像重要性 -> 技术路线比较 -> 选择 CNN -> CNN 临床应用挑战 -> 三个挑战递进 -> 三个对应方法 -> 实验验证。

三个挑战顺序固定为：

1. 多标签疾病共存。
2. 多机构数据难联合训练与跨机构部署域偏移并存，引出单域泛化（SDG）。
3. 跨域场景下的医疗数据隐私保护。

三个贡献顺序建议固定为：

1. ARC：自适应标签-特征关联性构建，解决多疾病共存下多标签识别问题。
2. DFE/SDG：双分支引导域不变特征提取，在多源 DG 难以实施而跨机构部署仍存在域偏移的场景下，仅依赖单一源域学习可迁移表示，并增强可解释性。
3. AdaGDP：自适应高斯差分隐私，解决隐私保护与性能平衡问题。

注意：贡献叙事顺序可以是 ARC -> DFE -> AdaGDP，但方法实现顺序可以保留为 DFE -> ARC -> AdaGDP。因为 ARC 在公式实现中依赖 DFE 输出的域不变特征。方法章节不需要防御式解释顺序差异，而应强调三个挑战在真实部署中构成耦合问题，框架按信息流组织各模块并实现协同。

### 6.1 待确认术语

**DFE 模块命名暂存疑。**

当前正文暂时使用“双分支引导域不变特征提取（Dual-branch guided domain invariant feature extraction, DFE）”。但严格来说，“域不变”通常需要更明确的域间度量或约束，例如 MMD 等机制支撑。本文该模块更想表达的是：在单一源域条件下，通过原图/增强图双分支和 CAM 引导，学习更可迁移的疾病相关表示，或提取更具泛化能力的疾病特征。

后续需与导师确认是否将 `domain-invariant feature` 调整为更保守、更贴合实验支撑的表述，例如：

- `generalizable feature`：强调疾病特征的泛化能力。
- `transferable feature`：强调疾病相关表示可迁移到未知域。

候选模块命名：

- 双分支引导泛化特征学习（Dual-branch guided Generalizable Feature Learning, DGL）。
- 双分支引导可迁移特征学习（Dual-branch guided Transferable Feature Learning, DTL）。
- 保留当前 DFE，但正文中弱化“域不变”的理论含义，更多使用“可迁移疾病相关表示”“疾病泛化特征”等描述。

避免使用的术语：

- 原则上避免在正文中使用“鲁棒性”“鲁棒特征”“鲁棒决策边界”等表述。该词含义较宽，容易引出对扰动类型、理论定义和额外鲁棒性实验的追问。
- 若确实需要表达模型在域偏移下表现更好，优先使用可由本文实验直接支撑的表述。

推荐替代表述：

- “泛化性能”：用于概括模型在同域/跨域测试中的整体表现。
- “跨域稳定性”：用于描述模型在不同数据集或不同机构条件下性能下降较小。
- “未知域性能”或“未知测试域性能”：用于描述 SDG 设置下未参与训练的数据集表现。
- “分布偏移下的性能保持能力”：用于替代更宏大的“鲁棒性”。
- “可迁移的疾病相关特征”：用于替代“鲁棒特征”。
- “更适用于未知域分布的决策边界”或“更稳定的决策边界”：用于替代“鲁棒决策边界”。
- “减少域特定噪声干扰”：用于具体说明方法作用，优先于笼统说“提升鲁棒性”。
